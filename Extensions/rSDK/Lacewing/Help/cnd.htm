On error:
On error is TRUE whenever an error occurs. Use the Error$("Lacewing") expression to get a description of the error.
Parameters: None.

Connect»On connect:
This condition is TRUE when a new connection has been made to a server, using the Connect action.
Parameters: None.

Connect»On connection denied:
When a connection, triggered by the Connect action, is refused by the server, this condition is TRUE. Use the DenyStatus$("Lacewing") and/or the Error$("Lacewing") expressions to find out why you were disconnected.
Parameters: None.

Connect»Is connected:
This condition is TRUE while a connection to a server, caused by the Connect action, is still active.
Parameters: None.

Connect»On disconnect:
This condition is TRUE when the connection to a server is closed. The connection is usually closed when you use the Disconnect action.
Parameters: None.

Channel listing»On List Received:
When the server replies with the list of channels, requested by the action Channel listing»Request list. Use the Channel Listing»Loop Channel List action to loop through the list.
Parameters: None.

Channel listing»On loop:
When you use the Channel Listing»Loop Channel List action, it starts a loop of all the items on the list. This condition is TRUE for each item on the list. Use the ChannelListing_Name$("Lacewing") expression to get channel name that is being looped through. You can also use the ChannelListing_PeerCount("Lacewing") expression to get the number of peers on the channel being looped.
Parameters: None.

Channel listing»On loop finished:
When you use the Channel Listing»Loop Channel List action, it starts a loop of all the items on the list. This condition is TRUE after the loop finishes.
Parameters: None.

Channel listing»[With loop name] On loop:
When you use the Channel Listing»Loop listed channels (with loop name) action, it starts a list loop, with whatever name you choose. This condition is TRUE for each item on the channel list. Use ChannelListing_Name$("Lacewing") expression to get channel name that is being looped through. You can also use ChannelListing_PeerCount("Lacewing") to get the number of peers on the channel being looped.
Parameters: The name of the loop (string).

Channel listing»[With loop name] On loop finished:
When you use the Channel Listing»Loop listed channels (with loop name) action, it starts a list loop, with whatever name you choose. This condition is TRUE once after a named Channel List loop finishes.
Parameters: The name of the loop (string).

Channel»On join:
This condition is TRUE each time you successfully join a channel. Channel_Name$("Lacewing") is automatically set to the name of the channel you just joined.
Parameters: None.

Channel»On join denied:
When a channel join request, triggered by the Channel»Join action, is refused by the server, this condition is TRUE. Use the DenyStatus$("Lacewing") and/or the Error$("Lacewing") expressions to find out why joining the channel was refused.
Parameters: None.

Channel»On leave:
This condition is TRUE when you leave a channel. Usually, this is caused by the Channel»Leave action.
Parameters: None.

Channel»On leave denied:
When a channel leave request, triggered by the Channel»Leave action, is refused by the server, this condition is TRUE once. Use the DenyStatus$("Lacewing") and/or the Error$("Lacewing") expressions to find out why leaving the channel was refused.
Parameters: None.

Channel»On joined channel loop:
When you use the Channel Listing»Select»Loop action, it starts a loop of all the channels you are joined to. This condition is TRUE for each channel you are joined to. Use the Channel_Name$("Lacewing") expression to get channel name that is being looped through. You can also use the Channel_PeerCount("Lacewing") expression to get the number of peers on the channel being looped.
Parameters: None.

Channel»On joined channel loop finished:
When you use the Channel Listing»Select»Loop action, it starts a loop of all the channels you are joined to. When the loop finishes, this condition will be TRUE.
Parameters: None.

Channel»[With loop name] On joined channel loop:
When you use the Channel Listing»Select»[With loop name] Loop action, it starts a loop of all the channels you are joined to with the name given. This condition is TRUE for each channel you are joined to. Use the Channel_Name$("Lacewing") expression to get channel name that is being looped through. You can also use the Channel_PeerCount("Lacewing") expression to get the number of peers on the channel being looped.
Parameters: The name of the loop (string).

Channel»[With loop name] On joined channel loop finished:
When you use the Channel Listing»Select»[With loop name] Loop action, it starts a loop of all the channels you are joined to with the name given. When the named loop finishes, this condition will be TRUE.
Parameters: The name of the loop (string).

Channel»Peers»On peer connect:
When a peer joins a channel you are joined to, this condition will be TRUE. Use Peer_Name$("Lacewing") to get the joining peer's name. Alternatively, use Peer_ID("Lacewing") to get the joining peer's ID.
Parameters: None.

Channel»Peers»On peer disconnect:
When a peer leaves a channel you are joined to, this condition will be TRUE. Use Peer_Name$("Lacewing") expression to get the leaving peer's name. Alternatively, use the Peer_ID("Lacewing") expression to get the leaving peer's ID.
Parameters: None.

Channel»Peers»On peer changed name:
When a peer changes his name, by using the Set Name action, this condition will be TRUE. Use Peer_PreviousName$("Lacewing") to get the old name and Peer_Name$("Lacewing") to get the new name (or use Peer_ID("Lacewing") to get the unchanged ID).
Parameters: None.

Channel»Peers»On loop:
When you use the Channel»Peers»Select»Loop action, it starts a loop of all the peers in the current channel (use the Channel_Name$("Lacewing") expression to get the current channel's name). This condition is TRUE for each peer on the channel. Use the Peer_Name$("Lacewing") or the Peer_ID("Lacewing") expressions to get the currently looped peer.
Parameters: None.

Channel»Peers»On loop finished:
When you use the Channel»Peers»Select»Loop action, it starts a loop of all the peers in the current channel. When the loop finishes, this condition will be TRUE.
Parameters: None.

Channel»Peers»[With loop name] On loop:
When you use the Channel»Peers»Select»[With loop name] Loop action, it starts a loop of all the peers in the current channel with the loop name given (use Channel_Name$("Lacewing") to get the current channel's name). This condition is TRUE for each peer on the channel. Use the Peer_Name$("Lacewing") or the Peer_ID("Lacewing") expressions to get the currently looped peer.
Parameters: The name of the loop (string).

Channel»Peers»[With loop name] On loop finished:
When you use the Channel»Peers»Select»[With loop name] Loop action, it starts a loop of all the peers in the current channel with the loop name given. When the named loop finishes, this condition will be TRUE.
Parameters: The name of the loop (string).

Channel»Peers»Selected peer is the channel master:
If the selected peer is the channel master (the first user to join the channel and create it), this condition will be TRUE. Use Peer_Name("Lacewing") or Peer_ID("Lacewing") to find out which peer is selected.
Parameters: None.

Channel»You are the channel master:
If you are the channel master for the current channel (the first user to join the channel and create it), this will be TRUE. Use Channel_Name$("Lacewing") to find out what channel is selected.
Parameters: None.

Name»On name set:
When you first successfully set your name using the Set Name action, this condition will be TRUE. If you already have a name, the Name»On name changed condition will be TRUE instead.
Parameters: None.

Name»On name denied:
When a name set request, triggered by the Name Set action, is refused by the server, this condition is TRUE. Use the DenyStatus$("Lacewing") and/or the Error$("Lacewing") expressions to find out why your name set was denied.
Parameters: None.

Name»On name changed:
If you use the Set Name action when you already have a name, and your name is changed successfully to a different name, this will be TRUE. If you don't have a name already, the Name»On name set condition will be TRUE instead.
Parameters: None.

Name»Client has a name:
If your client has a name, set by the Set Name action, this condition is TRUE. You can retrieve your name with the expression Self_Name$("Lacewing").
Parameters: None.
