Connect:
This action initiates a connection to a Lacewing server. If you use the "servername:serverport" syntax in the Host parameter, Lacewing will use the number after the ":" as the port. If you do not specify the port or use an invalid port number, Lacewing will use port 6121 by default. After the connection, you can use the HostIP$("Lacewing") expression to get the server's IP (not the name) or HostPort("Lacewing") expression to get the port.
Parameters: Host (string, servername, with optional ending of :serverport).

Disconnect:
This action disconnects the connection to the currently connected Lacewing server. You can use the HostIP$("Lacewing") expression to get the last server's IP (not the name) and HostPort("Lacewing") to get the last used port.
Parameters: None.

Set name:
This actions sets (or changes) the name of your client. If the name set/change was successful, the condition Name»On name set is TRUE. If it fails, the condition Name»On name set denied is set to TRUE and the DenyReason$("Lacewing") expression will explain why the action failed.
Parameters: New name (string).

Channel listing»Request list:
This action asks the server for it's list of channels. All the channels that are created with Join Channel action, with the Hide channel from list" parameter set to 1 will be on the list.
Parameters: None.

Channel listing»Loop list:
:list of channels received by the server. You should use it after the Channel listing»On list received condition is triggered. This will set the condition Channel listing»On loop TRUE for each channel on the list.
Parameters: None.

Channel listing»Loop list (with loop name):
:list of channels received by the server, with the chosen loop name. You should use it after the Channel listing»On list received condition is triggered. This will set the condition Channel listing»[With name]On loop TRUE for each channel on the list.
Parameters: Loop name (string).

Channel»Select»By name:
This action selects a channel that you are joined to by name. If you are joined to the channel, the Channel_Name$("Lacewing") expression is set to the name you chose. All Send/Blast to channel actions will be received by that channel. If you aren't joined to the channel, the Channel_Name$("Lacewing") expression will be set to an empty string ("").
Parameters: Channel name to select (string).

Channel»Select»Loop:
This action loops through the list of channels you are joined to. This will set the condition Channel»On joined channel loop TRUE for each channel you are joined to. When the loop finishes, it will set the condition Channel»On joined channel loop finished TRUE.
Parameters: None.

Channel»Select»Loop (with loop name):
:list of channels you are joined to, with the chosen loop name. This will set the condition Channel»[With name]On joined channel loop TRUE for each channel on the list. When the loop finishes, it will set the condition Channel»[With name]On joined channel loop finished TRUE.
Parameters: Loop name (string).

Channel»Join:
This action requests the server for access to a channel. If the server accepts your request, the condition Channel»On join will be TRUE. If the server refuses it, the condition Channel»On join denied will be TRUE. Use the Hide channel from list parameter to decide if the channel will be added to the list (the list requested in the action Channel listing»Request list) - if the channel was empty before you joined. Use the Close channel on leave parameter to tell the server if it should close the channel (remove all other peers joined to the channel) when you leave the channel if you the channel was empty before you joined (i.e. you created it).
Parameters: Channel name (string), Hide channel from list (number, 0 or 1), Close channel on leave (number, 0 or 1).

Channel»Leave:
This action closes your access to the current channel. Use the Channel»Select»By name action to select the channel before you use this action. The Channel_Name$("Lacewing") expression contains the name of the current channel.
Parameters: None.

Channel»Peer»Select by name:
This action selects a peer on the current channel by name. Any Send/Blast to Peer messages will go to that peer. Use Peer_ID("Lacewing") to get the currently selected peer's ID. Use Channel_Name$("Lacewing") to get the currently selected channel.
Parameters: Peer name to select (string).

Channel»Peer»Select by ID:
This action selects a peer on the current channel by ID. Any Send/Blast to Peer messages will go to that peer. Use Peer_Name$("Lacewing") to get the currently selected peer's name. Use Channel_Name$("Lacewing") to get the currently selected channel.
Parameters: Peer ID to select (number).

Channel»Peer»Select the channel master:
This action selects the current channel master. Any Send/Blast to Peer messages will go to that peer. Use Peer_Name$("Lacewing") or Peer_ID("Lacewing") after this action is run to get the channel master. Use Channel_Name$("Lacewing") to get the currently selected channel.
Parameters: None.

Channel»Peer»Loop:
This action loops through all the peers the current channel has. This will set the condition Channel»Peers»On loop TRUE for each channel you are joined to. When the loop finishes, it will set the condition Channel»On joined channel loop finished TRUE.
Parameters: None.

Channel»Peer»Loop (with loop name):
This action loops through all the peers the current channel has, with the chosen loop name. This will set the condition Channel»Peers»[With name] On loop TRUE for each peer on the current channel. When the loop finishes, it will set the condition Channel»Peers»[With name] On loop finished TRUE.
Parameters: Loop name (string).

Binary to send»Add byte»ASCII character:
This action appends (adds to the end) a ASCII character, (unsigned byte) e.g. "a", "1", "!" to the send binary. You can later transmit the send binary with any Send or Blast Binary action.
Parameters: Character to add (string).

Binary to send»Add byte»Integer value:
This action appends (adds to the end) a byte to the send binary. If your number can be negative, the range of values is -128 to +127. You should read it using the SignedByte("Lacewing") expression. You can later transmit the send binary with any Send or Blast Binary action.
Parameters: Byte to add (number between -128 and +127, or 0 and +255).

Binary to send»Add short:
This action appends (adds to the end) a short (two-byte number) to the send binary. If your number can be negative, the range of values you can send is -32,768 to +32,767. You should read it using the SignedShort("Lacewing") expression. If the number can be only positive, the range of values you can send is 0 to +65,535. You should read it using the Short("Lacewing") expression.
Parameters: Short to add (number between -32,768 and +32,767, or 0 to +65,535).

Binary to send»Add integer:
This action appends (adds to the end) a integer (four-byte number) to the send binary. If your number can be negative, the range of values you can send is -2,147,483,648 to +2,147,483,647. You should read it using the SignedShort("Lacewing") expression. If the number can be only positive, the range of values you can send is 0 to +4,294,967,295. You should read it using the Short("Lacewing") expression.
Parameters: Integer to add (number between -2,147,483,648 and +2,147,483,647 or 0 to +4,294,967,295).

Binary to send»Add float:
This action appends (adds to the end) a float (four-byte number with a decimal place) to the send binary. The range of values you can send is 3.4e-38 to +3.4e+38. You should read it using the Float("Lacewing") expression. This is the only data type you can use to add a number with a decimal place to the binary.
Parameters: Float to add (number between 3.4e-38 and 3.4e+38).

Binary to send»Add string»Without null terminator:
This action appends (adds to the end) a string to the send binary, without a null terminator byte at the end, for the StringWithSize&("Lacewing") expression. This takes up one byte more than your string length, but the receiver would have to be able to determine the length of the string when the client receives it, automatically.
Parameters: String to add (text of any length).

Binary to send»Add string»With null terminator:
This action appends (adds to the end) a string to the send binary, with a null terminator byte at the end which indicates the end of the string, for the String&("Lacewing") expression.
Parameters: String to add (text of any length).

Binary to send»Add binary:
This action appends (adds to the end) binary data from the memory, reading from the memory address number you supply, and reading the number of bytes you supply, and adding it to the send binary. You can read this from the memory using the BinaryAddress("Lacewing") expression.
Parameters: Memory address to read from (number), number of bytes to read (number).

Binary to send»Add file:
This action appends (adds to the end) all the binary data from a file to the send binary. You can read this from the memory using the BinaryAddress("Lacewing") expression. If you are appending a large file, the sender and/or receiver may stall briefly. Consider using the Send binary»Compress (ZLIB) action to make your data smaller before sending, and use the Received binary»Uncompress (ZLIB) action to return the data to its original after receiving.
Parameters: File name (string, file path).

Binary to send»Compress (ZLIB):
This action compresses all the data in your send binary, making it smaller and faster to transmit. Use the Received binary»Uncompress (ZLIB) action after receiving compressed data to return the data to its original. Note: You must do this before you do any other actions with the Lacewing received binary, as the data will not be the same as before you compressed it.
Parameters: None.

Binary to send»Clear:
This action empties your send binary. All data contained in your send binary is erased. Note: The data is automatically erased after you send if you have the Automatically clear binary property enabled.
Parameters: None.

Binary to send»Resize:
This action resizes your send binary, allowing you to copy a portion of memory directly to the send stack without using a Lacewing action. An object capable of doing this is the Memory object. You should use the SendBinarySize("Lacewing") expression for the size of the send binary and SendBinaryAddress("Lacewing") expression for the memory address of the send binary. Note: The data is automatically erased after you send if you have the Automatically clear binary property enabled.
Parameters: New size (number).

Received binary»Save to a file:
This action saves data from the binary that you just received into a file, overwriting any data already in the file. The position parameter is the 0-based start position, and the length is the number of bytes to read and save into the file.
Parameters: Position (number), Length (number), File name (string, filepath).

Received binary»Append to a file:
This action appends (adds to the end) data from the binary that you just received into a file, adding after all the original file data. The position parameter is the 0-based start position, and the length is the number of bytes to read and append to the file.
Parameters: Position (number), Length (number), File name (string, filepath).

Received binary»Uncompress (ZLIB):
This action restores compressed data to it's original, making it larger. This should be used by the receiver immediately after receiving if the sender has used the Send binary»Compress data (ZLIB) action.
Parameters: None.

Received binary»Move cursor:
This action moves the cursor of the received binary to the position you chose. Any expressions that use the cursor then read from that position. When you receive a new binary message, the cursor's position is set to 0.
Parameters: Position (number).
